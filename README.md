# Healz Mobile

Кроссплатформенная мобильная оболочка над публичным
[`app.healz.ai`](https://app.healz.ai) с нативным импортом и локальной
подготовкой медицинских документов.

Сейчас реализованы WebView shell, Share → Healz и локальная подготовка
документов. Приложение принимает PDF/изображения из Android Share Sheet,
копирует их во внутренний cache, аккуратно уменьшает размер и передаёт
результат штатному file input публичного сайта.

## Запуск

```bash

```

Для запуска на подключённом Android-телефоне:

```bash

npm run android
```

## Поведение WebView

- навигация внутри `app.healz.ai` остаётся в приложении;
- внешние HTTPS-ссылки, телефон, почта и карты открываются системой;
- Android Back возвращает по истории WebView;
- cookie, cache и DOM storage не очищаются между запусками;
- при сетевой ошибке показывается нативный экран с Retry.

Сохранение OTP-сессии после force-stop и перезагрузки необходимо подтвердить
на физическом телефоне.

## Отправка документа из другого приложения

1. Откройте PDF или изображение в Files, Gmail или галерее.
2. Нажмите «Поделиться» и выберите Healz Mobile.
3. Дождитесь локальной подготовки и проверьте размер до/после.
4. Подтвердите файл на нативном экране.
5. Если требуется, войдите в Healz по email-коду.
6. Приложение передаст файл в штатный uploader чата.

Подготовка выполняется только в sandbox приложения:

- большие изображения уменьшаются максимум до 2800 px по длинной стороне;
- JPEG сохраняется с качеством 0,86;
- результат изображения используется только при экономии не менее 10%;
- PDF не растрируется целиком: пережимаются только крупные встроенные
  изображения, сохраняя текстовые и векторные слои;
- PDF повторно открывается и проверяется на сохранение количества страниц;
- при ошибке, шифровании или невыгодном результате отправляется оригинал;
- временные оригиналы и подготовленные копии удаляются после отмены/отправки.

## Устойчивость

- до пяти документов обрабатываются и передаются строго по одному;
- в памяти одновременно находится base64 только текущего файла;
- после ошибки или ухода приложения в background очередь продолжает работу с
  первого ещё не отправленного файла;
- перед каждым файлом повторно проверяется подключение к интернету;
- при offline подготовленные документы остаются в cache до Retry или отмены;
- старый cache предыдущего аварийного запуска очищается при старте;
- нативные экраны поддерживают светлую/тёмную тему, safe areas, небольшие
  экраны и accessibility progress announcements;
- `npm run privacy-check` запрещает debug-логи в коде приложения и Kotlin-модуле.

Ограничения прототипа:

- PDF, JPEG, PNG, WebP, HEIC и HEIF;
- максимум 5 файлов;
- максимум 20 МБ суммарно, как в публичной конфигурации Healz;
- фактическую загрузку нужно подтвердить на физическом телефоне.

## APK

Локально собранный автономный APK для ARM64 находится в
`dist/Healz-Mobile-8.5-arm64.apk`. Папка `dist` исключена из Git.

Установка на подключённый телефон:

```bash
adb install -r dist/Healz-Mobile-8.5-arm64.apk
```

Это прототипная Gradle release-сборка со встроенным JS-бандлом. APK подписан
локальным Android debug certificate, чтобы его можно было поставить напрямую и
обновить поверх предыдущего локального прототипа. Для продакшен-релиза нужен
постоянный release/upload key или EAS-managed credentials.

SHA-256:

```text
2a8103b1570ea4e713ecbe07377ae0e518c188076a189b1afd64b80105c49313
```

Тот же хэш лежит рядом: `dist/Healz-Mobile-8.5-arm64.apk.sha256`.

Перед первой сборкой необходимо вручную войти в Expo:

```bash
eas login
eas whoami
```

Профиль `preview` уже настроен на выпуск устанавливаемого APK:

```bash
eas build --platform android --profile preview
```
