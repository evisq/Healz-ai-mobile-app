# Acceptance checklist

Это проверяемые критерии результата. Пункт отмечается выполненным только после
проверки на физическом Android-телефоне.

## Этап 0 — подготовка

- [x] Создан Expo SDK 56 проект с TypeScript.
- [x] Настроены идентификаторы приложения и APK build profile.
- [x] Зафиксированы критерии приёмки и способ проверки.
- [x] Подготовлены синтетические PDF/JPEG/PNG без персональных данных.
- [x] Настроен GitHub-репозиторий.
- [ ] Выполнен ручной вход в Expo/EAS.
- [ ] Физический Android-телефон виден в `adb devices -l`.

## WebView

Реализация WebView shell завершена. Чекбоксы ниже остаются пустыми до ручной
проверки на физическом Android-телефоне.

- [ ] `https://app.healz.ai` открывается внутри приложения.
- [ ] Работает email + шестизначный OTP-вход.
- [ ] Сессия сохраняется после обычного перезапуска.
- [ ] Сессия сохраняется после force-stop.
- [ ] Сессия сохраняется после перезагрузки телефона.
- [ ] Android Back возвращает по истории WebView.
- [ ] При пустой истории Back закрывает/сворачивает приложение штатно.
- [ ] Внутренние ссылки Healz остаются в WebView.
- [ ] Внешние HTTPS, `mailto:` и `tel:` открываются системными приложениями.
- [ ] Есть собственные icon и splash.
- [ ] Ошибка сети имеет понятный Retry.

## Импорт документов

Код Share Intent, cache-copy и WebView bridge реализован. Чекбоксы остаются
пустыми до end-to-end проверки на физическом Android-телефоне и реальном
Healz-аккаунте.

- [ ] Healz виден в Android Share Sheet для `application/pdf`.
- [ ] Healz виден в Android Share Sheet для `image/*`.
- [ ] PDF принимается из Files.
- [ ] Изображение принимается из Google Photos/галереи.
- [ ] Документ принимается из почтового приложения.
- [ ] Временный `content://` URI безопасно копируется в app cache.
- [ ] Пользователь видит имя, тип и размер файла до подтверждения.
- [ ] Отмена не загружает файл и очищает временные данные.
- [ ] Подготовленный файл реально появляется в штатном uploader чата Healz.
- [ ] Файл успешно загружается в чат.

## Подготовка файлов

Image/PDF pipelines, fallback на оригинал, сравнение размеров и cleanup
реализованы. Чекбоксы остаются пустыми до проверки качества на физическом
телефоне; Kotlin-модуль отдельно проходит компиляцию.

- [ ] Всё сжатие выполняется локально на устройстве.
- [ ] Исходный файл не изменяется и не удаляется.
- [ ] JPEG уменьшается без потери читаемости мелкого текста.
- [ ] PNG не конвертируется в худший по качеству или больший файл.
- [ ] PDF сохраняет количество страниц.
- [ ] PDF сохраняет текстовый слой, когда он был в оригинале.
- [ ] Результат используется только если он валиден и меньше оригинала.
- [ ] При ошибке доступна отправка оригинала.
- [ ] В логах нет содержимого документов, OTP и auth-cookie.

## Устойчивость и polish

Код очереди, offline/retry, lifecycle, accessibility и адаптивного нативного
UI реализован. Визуальные пункты остаются для проверки на телефоне.

- [ ] Пять файлов подготавливаются и передаются последовательно.
- [ ] После ошибки Retry не отправляет уже добавленные файлы повторно.
- [ ] Потеря сети сохраняет подготовленные документы и показывает Retry.
- [ ] Возврат из background продолжает очередь с нужного файла.
- [ ] Старые временные файлы очищаются при следующем запуске.
- [ ] Bottom sheet прокручивается на небольшом экране.
- [ ] Клавиатура не перекрывает системные сообщения приложения.
- [ ] UI корректен в светлой и тёмной теме.
- [ ] TalkBack объявляет подготовку, прогресс передачи и offline.
- [x] `npm run privacy-check` проходит.
