# Журнал работы AI-агента

## 2026-06-25 — этап 0

### Задача

Подготовить среду, GitHub-репозиторий и минимальный Expo-проект до начала
реализации WebView и нативного импорта.

### Решения

- Выбран Expo SDK 56 + React Native + TypeScript.
- Android package: `com.evisq.healzaimobile`.
- Добавлен EAS-профиль `preview` с `buildType: apk`.
- Эмулятор исключён: пользователь будет проверять приложение на физическом
  телефоне, что особенно важно для Share Intent.
- Настоящие медицинские документы запрещены; fixtures генерируются локально и
  содержат только вымышленные значения.

### Проверка

- `npm install`
- `npm run typecheck`
- `npm run doctor`
- проверка Android SDK, JDK и ADB;
- визуальная и техническая проверка синтетических fixtures.

### Ограничения

- Expo/EAS требует ручной интерактивный вход пользователя.
- Телефон ещё должен быть подключён и авторизован через ADB.
- WebView и Share Intent относятся к следующим этапам и пока не реализованы.
